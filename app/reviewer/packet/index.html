審閱入口

證據審閱資料包

依合成案例證據判斷流程要繼續、修正或停止；判斷依據是流程價值與安全邊界，不是展示包裝。

僅使用合成資料。 不做診斷、分流或治療建議。 仍需醫療人員確認。

審閱順序

請依序檢查

  1. 開啟病人端。
  2. 載入合成案例。
  3. 檢查護理缺漏補問。
  4. 檢查醫師摘要與看診前資料包。
  5. Run `npm run experiment:phase1`.
  6. 記錄審閱回饋並完成評分表。

立即停止條件

若任何畫面暗示診斷、分流、治療、真實病人資料使用，或把缺漏資訊藏起來，請先修正再繼續。
